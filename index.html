Installation

Follow the steps below to setup your colrows:

  1. Unzip the downloaded package and extarct the dist-<version>.tar to find all the template files.
  2. The setup is already equipped with minimum configuration required to kick start
    • open bin folder; and run colrows.sh
    • navigate to http://localhost:80/login

Docker Installation

Colrows provide exclusive support docker, in case you dont have docker installed, please use the Docker Installation Guide

  1. use given command to pull latest colrows image : docker pull colrows/colrows:latest
  2. use given command to run recently pulled colrows image : docker container run -it -d –name colrows-contaner -p 80:80 colrows/colrows:latest
  3. open any browser and go to http://localhost:80/login

AWS Cloud Installation

To kick start your colrows experience on Amazon Web Services Cloud follow the below given steps:

  1. go to AWS portal and login with your AWS Credentials
  2. in top search bar type cloud formation and select CloudFormation option

  3. On CloudFormation portal, click on Stacks from left hand side menu option and select Create Stack

  4. Enter the details on Create Stack page
    • From Prerequisit - Prepare Template select Template is ready
    • From Specify template select Upload a tempalte file
    • Click on Choose file and upload aws-cloud-formation-template.yaml (you must have get it with your colrows subscription)
    • Click Next

  5. Enter the details on Specify stack details page
    • In Stack name provide the name of the stack
    • Select availability Zone
    • In Instance type select your preferred instance type, t2.micro is sufficient for exploring colrows
    • In AWSKey section select Amaxon EC2 Key Pair; in case you dont have one, create it from EC2 service -> Network & Security -> Key Pairs
    • Click Next

  6. on Configure stack option, configure all option as per your need (you can ignore this in basic setup), and click Next
  7. on Review window, review all the configuration and click on Create, it will build the complete stack of required components for you
  8. to observe resources created by template navigate to resources tag

  9. once stack creation completed, assure EC2 instance created and its health check is completed. (colrows would have approx 5 min to install post EC2 instance is ready)

  10. click on EC2 instance ID to goto EC2 instance propertues section

  11. observe public IP of your EC2 instance and access http://<your-public-ip>:80/login
  12. your colrows setup is completed, and you are good explore.

Microsoft Azure Installation

To kick start your colrows experience on Microsoft Azure Cloud follow the below given steps:

  1. go to Azure template service : https://portal.azure.com/#create/Microsoft.Template and select Build your own template in the editor option

  2. Click on Load file and upload azure-resource-manager-template.yaml (you must have get it with your colrows subscription)

  3. it will create all required resources including Network Security Group, Public IP Address, Virtual Network, Virtual Machine, etc; click save to proceed

  4. Provide all the required details to create resource dynamically

  5. once all resources created, look for public IP of the VM created, goto http://<your-public-ip>:80/login
  6. your colrows setup is completed, and you are good explore.

Google Cloud Installation

Unlike AWS and Azure cloud, Google doesnt provide UI based approach to create resource dynamically using templates. we will use gcloud CLI instead

  1. install Google Cloud CLI refer : Google cloud CLI install Guide
  2. Initialize and Authenticate google cloud CLI; and create new project using gcloud init refer : Configure Google cloud CLI
  3. Enable HTTP and HTTPS traffic for the newly created project using command :
    gcloud compute firewall-rules create http-allow --allow tcp:80,tcp:443
  4. Create new deployment using tempalte google-deployment-manager-template.yaml (you must have get it with your colrows subscription)
  5. command to create deployment using template :
    gcloud deployment-manager deployments create colrows --config google-deployment-manager-template.yaml
  6. monitor resource created by the template, goto "Deployment Manager" and assure deployment creation compelted succesfully.

  7. once deployment is successful, look for public IP of your VM created by Deployment and goto http://<your-public-ip>:80/login

  8. it may take 5 min to colrows available on public IP, as colrows will perform initial setup in background for first time
  9. your colrows setup is completed, and you are good explore.

Configuration

below is the configuration details to configure/setup different features provided by ColRows

HTTP/HTTPS Configuration

To enable HTTPS support update conf/server.yaml and update below configuration:

HTTP Configuration


            http-server:              
              connector:
                http:
                    port: <int> # port to serve HTTP traffic; default port is 80

HTTPS Configuration

update conf/server.yaml and update below configuration with respective values


            http-server:              
              connector:
                https:
                    enable: <boolean> # true/false indicating SSL enable status
                    port: <int> # port to serve HTTPS traffic; default port is 443
                    ssl:
                      self-signed: <boolean> # flag to indicate if self signed certificate is used                      
                      key-store: <string> # Path of keystore where PKCS12 file is imported;  
                      # in case you have copied keystore in conf folder; only file name will also work (most probably ketstore)
                      key-store-password: <string> # keystore password
                      key-manager-password: <string> # key manager password

Importing the PKCS12 file in keystore

Before proceeding make sure you have JAVA environement variable are set. keytool is available in java\bin location

  1. Execute the keytool import command :
    keytool -importkeystore -srckeystore colrows.pkcs12 -srcstoretype PKCS12 -destkeystore keystore
  2. Enter a password for the keystore (colrows in this example).
  3. Re-type the password.
  4. Enter the password you have defined before for the PKCS12 file (colrows).

Now we have created a new keystore and imported our PKCS12 file. The same keystore we will refer to enable SSL/HTTPS in HTTPS Configuration Section

Authentication

Colrows is equipped with all industry leading authentication mechanisms, along with BASIC Authentication (Default) which provides you admin access and let you configure Colrows as per your need.

Colrows allowed you to configure multiple authentication mechanisms as well (define as realm in colrows configuration), and it will utilise this multiple configured authentication techniques based on their configure precedence, defined as order while configuring.

LDAP Authentication

LDAP / Active Directory Authentication is one of the leading authetication mechanisms in the industry, colrows provide out of box integration with it, yet simple to configure.

edit yourconf/server.yaml and update below configuraiton :


      security:
        auth:
          realms:
            ldap:
              domain-name: <string> #domain name of your LDAP server
              url: <string> #domain end-point URL of your LDAP server
              bind-dn-username: <string> #OPTIONAL #previllaged bind account username to query on behalf of non-previllaged user
              bind-dn-password: <string> #OPTIONAL #previllaged bind account password to query on behalf of non-previllaged user
              order: <int> #order to consider in case multiple authentication mechanisms are configured. default : 999

OAuth/OIDC Authentication

Another very popular authentication mechanisms, specially used in third party authentication is OAuth which supported by Colrows as seemlessly as all other authentications

edit yourconf/server.yaml and update below configuraiton :


      security:
        auth:
          realms:
            oidc:
              provider: <string> #OAuth/OIDC Authentication provider
              client-id: <string> #client ID of application as configured in OAuth provider
              client-secret: <string> #client secret of application as provided in OAuth provider
              base-url: <string> #dEnd-point URL of OAUTH server
              realm: <string> #realm in which your application is configured in OAuth provider
              order: <int> #order to consider in case multiple authentication mechanisms are configured. default : 999

Colrows provide OAuth integration with Google, Azure and Keycloak. add provider in your server.yaml configuration as per your configuration requirement

  • GOOGLE
  • AZURE
  • KEYCLOAK (for keycloak server version above 17)
  • KEYCLOAK_LEGACY (for Keycloak server version 17 and older)

Session Yeilding

Colrows provide out of box feature of session yeilding, to assure all matured sessions are timely yield and backuped for analysis.

Colrows utilises own multithreaded environemnt to manage and run timely session monitoring job and extract all matured (non-active and older than 2 weeks) session to session.csv file in data location.
